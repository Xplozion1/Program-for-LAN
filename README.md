## Программа для подбора оптимальной конфигурации оборудования ЛВС

Приложение предназначено для **подбора оптимальной конфигурации оборудования локальной вычислительной сети (ЛВС)**.  
Пользователь задаёт требования к сети и весовые коэффициенты критериев, после чего программа подбирает оптимальные модели
коммутаторов, маршрутизаторов, IP‑камер, серверов и кабелей, а также позволяет сравнивать несколько полученных конфигураций.

### Основные возможности

- **Метод взвешенных сумм**  
  Подбор оптимального оборудования по нескольким критериям (скорость, цена, мощность, количество портов и т.д.) с учётом
  весов, задаваемых пользователем.

- **Симплекс‑метод**  
  Поиск оптимального оборудования как задачи линейного программирования с ограничениями по характеристикам
  (минимальная/максимальная цена, скорость, объём памяти и др.).

- **Генетический алгоритм**  
  Эволюционный поиск оптимальной модели с учётом ограничений и параметров алгоритма (размер популяции, число поколений,
  вероятность мутации и скрещивания) с подробным логом работы.

- **Работа с базой данных оборудования**  
  Все устройства (коммутаторы, маршрутизаторы, камеры, серверы, кабели) и их характеристики хранятся в локальной базе
  данных `DevicesDatabase` (SQLite).  
  Результаты оптимизаций записываются в таблицу `DevicesDatabase` с указанием:
  - номера конфигурации;
  - типа устройства;
  - текстового описания выбранной модели и её характеристик;
  - используемого метода оптимизации.

- **Просмотр найденных конфигураций**  
  Форма `Form5` отображает все найденные конфигурации, их технические характеристики и использованный метод оптимизации.

- **Сравнение конфигураций и расчёт стоимости**  
  Форма `Form6` позволяет выбрать несколько конфигураций, задать количество устройств (или длину кабеля) и
  автоматически рассчитать итоговую стоимость, определяя наиболее выгодную конфигурацию.

### Технологии и стек

- **Язык**: C#  
- **Платформа**: .NET 8.0 (Windows, WinForms)  
- **Графический интерфейс**: Windows Forms (`Form1`–`Form6`)  
- **База данных**: SQLite (БД в `DevicesDatabase`)  
- **Доступ к данным**: `System.Data.SQLite`  
- **Численные расчёты**: `MathNet.Numerics` (нормализация данных, работа с числовыми массивами и т.п.)  
- **Собственные реализации алгоритмов**:
  - метод взвешенных сумм;
  - симплекс‑метод;
  - генетический алгоритм (`GeneticAlgorithm` в `Form4.cs`).

### Структура проекта (основное)

- `Program.cs` – точка входа приложения.  
- `Form1` – главная форма, выбор метода оптимизации и переход к другим формам, инициализация работы с БД.  
- `Form2` – метод взвешенных сумм для всех типов устройств.  
- `Form3` – симплекс‑метод.  
- `Form4` – генетический алгоритм и подробный лог вычислений.  
- `Form5` – просмотр полученных конфигураций.
- `Form6` – сравнение конфигураций и расчёт итоговой стоимости.  
- `DevicesDatabase` – файлы базы данных SQLite с таблицами устройств и результатов оптимизации.

### Как запустить

1. Открыть решение `diplom.sln` в Visual Studio 2022 (или более новой версии) на Windows.  
2. Убедиться, что в каталоге проекта существует папка `DevicesDatabase` с файлами базы данных.  
3. Собрать проект.  
4. Запустить проект (`F5` или `Ctrl+F5`).

После запуска откроется главная форма, где можно выбрать метод оптимизации, задать параметры и сохранить/сравнить
полученные конфигурации.
